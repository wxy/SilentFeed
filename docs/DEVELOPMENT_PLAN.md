## 开发节奏建议

### 每个阶段的流程

1. **开始前** (5 分钟)
   - 阅读阶段文档
   - 创建功能分支: `git checkout -b feature/phase-N`

2. **开发中** (主要时间)
   - 编写测试 (TDD)
   - 实现功能
   - 频繁提交: `git commit -m "feat: xxx"`

3. **完成后** (10 分钟)
   - 运行测试: `npm test`
   - 检查覆盖率: `npm run test:coverage`
   - 更新文档
   - 合并到 master: `git merge feature/phase-N`
   - 推送: `git push`

### 每日开发建议

- **每次 1-2 小时**: 完成一个小功能
- **立即验证**: 在浏览器中测试
- **及时提交**: 不积累太多未提交代码
